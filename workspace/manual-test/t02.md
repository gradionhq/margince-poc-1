## 3. `Partner` schema — PO-DDL-6 target values present, additive-first (gate issue #58, Option 1)

Run: `grep -n "^    Partner:" backend/api/crm.yaml` then read that schema block.

Expected: `cert_status` enum **unions** the PO-DDL-6 target values (`applied`, `certified`,
`suspended`) with the legacy live values (`prospect`, `in_certification`, `churned`) — nothing
removed; `partner_role` is `type: string` with enum `[hosting, consulting, strategic]` and
carries `x-extensible-enum: true`; `margin_tier` is `type: [string, 'null']` with enum
`[null, tier1_15, tier2_20, tier3_25]` and carries `x-extensible-enum: true`. `gate_metrics`
is **still present** (kept, described as deprecated — not removed); `certified_staff` (integer),
`retention_rate` (nullable 0-100 number), `id`, `workspace_id`, `joined_at` (nullable date),
`renews_at` (nullable date) are all present alongside it.

Note: `partner_role` was kept non-nullable (type: string, not [string, 'null']) because oasdiff
classifies making a response property nullable as an ERR-level breaking change
(`response-property-became-nullable`). The fast-follow ticket should handle this alongside
the DB column change. Confirm the PR description names this gap.
