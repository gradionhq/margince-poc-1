# T16 Manual UAT - Composite 360 Reads

Run this only after the stack is up:

```bash
make infra-up && make migrate-up && make seed-reset && make run
```

The commands below assume:

- the API is listening on `http://localhost:8080`
- the database is reachable through `$DATABASE_URL`
- you run all steps in one shell session so exported variables stay available

Set these from `make seed-reset`'s output before running the guide for real:

```bash
export API_BASE='http://localhost:8080'
export WORKSPACE_ID='<seeded workspace id>'
export USER_ID='<seeded user id>'
export PIPELINE_ID='<seeded pipeline id>'
export STAGE_ID='<seeded stage id>'
```

## Step 1: Seed a person and relationship

```bash
export PERSON_ID="$(curl -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  -d '{"full_name":"T16 UAT Person","source":"manual-test","captured_by":"human:uat"}' \
  "$API_BASE/people" | jq -r '.id')"

export ORG_ID="$(curl -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  -d '{"display_name":"T16 UAT Org","source":"manual-test","captured_by":"human:uat"}' \
  "$API_BASE/organizations" | jq -r '.id')"

curl -i -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  -d "{\"kind\":\"employment\",\"person_id\":\"$PERSON_ID\",\"organization_id\":\"$ORG_ID\",\"source\":\"manual-test\",\"captured_by\":\"human:uat\"}" \
  "$API_BASE/relationships"
```

Expected: each create returns `201 Created`; record the returned person id as `$PERSON_ID` and org id as `$ORG_ID`, and the relationship links `$PERSON_ID` to `$ORG_ID`.

## Step 2: GET /people/{id} returns the person-360 composite

```bash
curl -s \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/people/$PERSON_ID" | jq '{relationships, deals, activities}'
```

Expected: `relationships` is a one-element array whose `organization_id` equals `$ORG_ID`; `deals` and `activities` are present as arrays.

## Step 3: GET /organizations/{id} returns the organization-360 composite

```bash
curl -s \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/organizations/$ORG_ID" | jq '{relationships, deals, activities}'
```

Expected: `relationships` is a one-element array whose `person_id` equals `$PERSON_ID`; `deals` and `activities` are present as arrays.

## Step 4: Seed a deal and stakeholder, then GET /deals/{id} returns the deal-360 composite

```bash
export DEAL_ID="$(curl -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  -d "{\"name\":\"T16 UAT Deal\",\"pipeline_id\":\"$PIPELINE_ID\",\"stage_id\":\"$STAGE_ID\",\"source\":\"manual-test\",\"captured_by\":\"human:uat\"}" \
  "$API_BASE/deals" | jq -r '.id')"

curl -i -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  -d "{\"kind\":\"deal_stakeholder\",\"deal_id\":\"$DEAL_ID\",\"person_id\":\"$PERSON_ID\",\"role\":\"champion\",\"source\":\"manual-test\",\"captured_by\":\"human:uat\"}" \
  "$API_BASE/relationships"

curl -s \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/deals/$DEAL_ID" | jq '{id, stakeholders, timeline}'
```

Expected: the deal create returns `201 Created` and a UUID that you record as `$DEAL_ID`; the stakeholder create returns `201 Created`; `GET /deals/{id}` returns a flat body with `id == $DEAL_ID`, one stakeholder for `$PERSON_ID` with role `champion`, and a `timeline` array.

## Step 5: Archived records stay fetchable through their composite

```bash
curl -s -X DELETE \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/people/$PERSON_ID"

curl -s \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/people/$PERSON_ID" | jq '{id, archived_at, relationships}'
```

Expected: the second call returns `200 OK` instead of `404`, `archived_at` is non-null, and `relationships` is still populated.

## Step 6: Unrelated rows never leak into the composite

```bash
curl -i -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  -d '{"full_name":"T16 Unrelated Person","source":"manual-test","captured_by":"human:uat"}' \
  "$API_BASE/people"

curl -s \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/organizations/$ORG_ID" | jq '.relationships | length'
```

Expected: the unrelated person is created successfully, but the organization composite still reports one relationship and does not include the unrelated person.

## Step 7: A caller with no visibility into the parent still gets a plain 404

```bash
curl -s -o /dev/null -w "%{http_code}" \
  -H "X-User-ID: $USER_ID" \
  -H "X-Workspace-ID: $WORKSPACE_ID" \
  "$API_BASE/people/00000000-0000-0000-0000-000000000099"
```

Expected: `404`, with no existence leak.
