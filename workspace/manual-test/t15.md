# T15 Live UAT Guide — Partner Registration + Deal `partner_org_id`

Run this only after the stack is up:

```bash
make infra-up && make migrate-up && make seed-reset && make run
```

The commands below assume:

- the API is listening on `http://localhost:8080`
- the database is reachable through `$DATABASE_URL`
- you run all steps in one shell session so exported variables stay available

## Bootstrap

Seed a workspace, two orgs (one for the partner steps, one with no partner row for the 404 step),
a pipeline + stage, two users, and two roles: one `t15-admin` with the `"partner"` RBAC key (used
for steps 1–4) and one `t15-rep` without it (used for the 403 RBAC gate step).

```bash
export API_BASE='http://localhost:8080'
export WS_ID='00000000-0000-0000-0000-000000000015'

# Fixed IDs so the bootstrap is idempotent (re-run safe).
export ADMIN_USER_ID='00000000-0000-0000-0015-000000000001'
export REP_USER_ID='00000000-0000-0000-0015-000000000002'
export ADMIN_ROLE_ID='00000000-0000-0000-0015-000000000010'

export PIPELINE_ID='00000000-0000-0000-0015-000000000030'
export STAGE_ID='00000000-0000-0000-0015-000000000031'

psql "$DATABASE_URL" -v ON_ERROR_STOP=1 <<'SQL'
INSERT INTO workspace (id, name, slug, base_currency)
  VALUES ('00000000-0000-0000-0000-000000000015', 'T15 UAT', 't15-uat', 'USD')

  VALUES ('00000000-0000-0000-0015-000000000031', '00000000-0000-0000-0000-000000000015', '00000000-0000-0000-0015-000000000030', 'T15 Stage', 1)
  ON CONFLICT (id) DO NOTHING;

INSERT INTO app_user (id, workspace_id, email, display_name)
  VALUES
    ('00000000-0000-0000-0015-000000000001', '00000000-0000-0000-0000-000000000015', 'admin-t15@example.com', 'T15 Admin'),
```

## Step 0 (RBAC gate): non-admin principal receives 403 on PUT

This proves the `"partner"` RBAC key gate is live before any data is created. The rep user has no
`"partner"` key in their permissions.

```bash
curl -i -sS -X PUT \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${REP_USER_ID}" \
  -d '{"partner_role":"hosting","cert_status":"applied","source":"uat","captured_by":"human:t15"}' \
  "${API_BASE}/organizations/${PARTNER_ORG_ID}/partner"
```

Expected: `403 Forbidden`. The response body contains a problem object; no partner row is created.

---

## Step 1: Create a partner row (first upsert)

Uses the admin principal. The `PARTNER_ORG_ID` org has no partner row yet.

```bash
curl -i -sS -X PUT \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  -d '{"partner_role":"hosting","cert_status":"applied","source":"uat","captured_by":"human:t15"}' \
  "${API_BASE}/organizations/${PARTNER_ORG_ID}/partner"
```

Expected: `200 OK`. Response body includes `"cert_status":"applied"` and `"partner_role":"hosting"`.
Record the returned `id` for step 2:

```bash
export PARTNER_ID="$(psql "$DATABASE_URL" -At -c \
  "SELECT id FROM partner WHERE organization_id='${PARTNER_ORG_ID}' AND workspace_id='${WS_ID}';")"
echo "PARTNER_ID=${PARTNER_ID}"
```

Verify the org's `classification` was set to `partner` in the same transaction:

```bash
psql "$DATABASE_URL" -c \
  "SELECT id, classification FROM organization WHERE id='${PARTNER_ORG_ID}' AND workspace_id='${WS_ID}';"
```

Expected: `classification = partner`.

Verify one `audit_log` row and one `partner.created` outbox row were written:

```bash
psql "$DATABASE_URL" -c \
  "SELECT action, entity_type FROM audit_log WHERE entity_type='partner' AND entity_id='${PARTNER_ID}' ORDER BY occurred_at DESC LIMIT 1;"

psql "$DATABASE_URL" -c \
  "SELECT topic FROM event_outbox WHERE topic='partner.created' AND entity_id='${PARTNER_ID}' ORDER BY id DESC LIMIT 1;"
```

Expected: `audit_log` row has `action='create'` and `entity_type='partner'`. `event_outbox` row has
`topic='partner.created'`.

## Step 2: Update the same partner row (second upsert — idempotency + update path)

The same org already has a partner row. The upsert must update it, not create a duplicate.

```bash
curl -i -sS -X PUT \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  -d '{"partner_role":"hosting","cert_status":"certified","source":"uat","captured_by":"human:t15"}' \
  "${API_BASE}/organizations/${PARTNER_ORG_ID}/partner"
```

Expected: `200 OK`. Response body includes `"cert_status":"certified"` and the same `"id"` as in step 1.

Verify only one partner row exists for this org (no duplicate):

```bash
psql "$DATABASE_URL" -c \
  "SELECT count(*) FROM partner WHERE organization_id='${PARTNER_ORG_ID}' AND workspace_id='${WS_ID}';"
```

Expected: `count = 1`.

Verify a second `audit_log` row was written (action `update`):

```bash
psql "$DATABASE_URL" -c \
  "SELECT action FROM audit_log WHERE entity_type='partner' AND entity_id='${PARTNER_ID}' ORDER BY occurred_at DESC LIMIT 1;"
```

Expected: `action = update`.

Total audit rows for this partner after steps 1+2:

```bash
psql "$DATABASE_URL" -c \
  "SELECT count(*) FROM audit_log WHERE entity_type='partner' AND entity_id='${PARTNER_ID}';"
```

Expected: `count = 2`.

## Step 4: List partners filtered by role and cert_status

Step 1/2 upserted a partner with `partner_role=hosting` and `cert_status=certified`. Only that row
should appear; `EMPTY_ORG_ID` has no partner row and must not appear.

```bash
curl -i -sS \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  "${API_BASE}/partners?partner_role=hosting&cert_status=certified"
```

Expected: `200 OK`. Response body is a page object (`{"data":[...],"next_cursor":...}`). The `data`
array contains at least one entry whose `organization_id` equals `${PARTNER_ORG_ID}` and no entry
whose `organization_id` equals `${EMPTY_ORG_ID}`.

Confirm a mismatched filter returns no results for this org:

```bash
curl -sS \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  "${API_BASE}/partners?partner_role=consulting&cert_status=certified"
```

Expected: `200 OK`. `data` array does not contain an entry with `organization_id=${PARTNER_ORG_ID}`
(it was seeded as `hosting`, not `consulting`).

## Step 5: Create a deal with `partner_org_id`

```bash
curl -i -sS -X POST \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  -d "{\"name\":\"T15 partner deal\",\"pipeline_id\":\"${PIPELINE_ID}\",\"stage_id\":\"${STAGE_ID}\",\"partner_org_id\":\"${PARTNER_ORG_ID}\",\"source\":\"uat\",\"captured_by\":\"human:t15\"}" \
  "${API_BASE}/deals"
```

Expected: `201 Created`. Response body includes `"partner_org_id":"<PARTNER_ORG_ID>"` with the
actual UUID value of `${PARTNER_ORG_ID}`.

Capture the deal ID for subsequent steps:

```bash
export DEAL_ID="$(psql "$DATABASE_URL" -At -c \
  "SELECT id FROM deal WHERE workspace_id='${WS_ID}' AND name='T15 partner deal' ORDER BY created_at DESC LIMIT 1;")"
echo "DEAL_ID=${DEAL_ID}"
```

Verify the persisted value in the database:

```bash
psql "$DATABASE_URL" -c \
  "SELECT id, partner_org_id FROM deal WHERE id='${DEAL_ID}' AND workspace_id='${WS_ID}';"
```

Expected: `partner_org_id` equals `${PARTNER_ORG_ID}`.

## Step 6: Update the deal's `partner_org_id` (attribution change + audit)

First, seed a second partner org to use as the new attribution target:

```bash
export PARTNER_ORG_B_ID='00000000-0000-0000-0015-000000000022'

psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c \
  "INSERT INTO organization (id, workspace_id, name, source, captured_by)
   VALUES ('${PARTNER_ORG_B_ID}', '${WS_ID}', 'T15 Partner Org B', 'uat', 'human:t15')
   ON CONFLICT (id) DO NOTHING;"
```

Upsert a partner row for org B so the FK target is valid:

```bash
curl -i -sS -X PUT \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  -d '{"partner_role":"consulting","cert_status":"applied","source":"uat","captured_by":"human:t15"}' \
  "${API_BASE}/organizations/${PARTNER_ORG_B_ID}/partner"
```

Expected: `200 OK`.

Now patch the deal to change its `partner_org_id` from org A to org B:

```bash
curl -i -sS -X PATCH \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  -d "{\"partner_org_id\":\"${PARTNER_ORG_B_ID}\"}" \
  "${API_BASE}/deals/${DEAL_ID}"
```

Expected: `200 OK`. Response body includes `"partner_org_id":"<PARTNER_ORG_B_ID>"` reflecting the
new value.

Verify the new value was persisted:

```bash
psql "$DATABASE_URL" -c \
  "SELECT partner_org_id FROM deal WHERE id='${DEAL_ID}' AND workspace_id='${WS_ID}';"
```

Expected: `partner_org_id` equals `${PARTNER_ORG_B_ID}`.

Verify exactly one `audit_log` row was written for this change capturing the prior value (S-E03.6):

```bash
psql "$DATABASE_URL" -c \
  "SELECT action, entity_type, before FROM audit_log
   WHERE entity_type='deal' AND entity_id='${DEAL_ID}' AND action='update'
   ORDER BY occurred_at DESC LIMIT 1;"
```

Expected: one row with `action='update'`, `entity_type='deal'`. The `before` JSONB column contains
`"partner_org_id":"<PARTNER_ORG_A_UUID>"` — the prior value (`${PARTNER_ORG_ID}`).

Verify exactly one `deal.partner_assigned` event was enqueued:

```bash
psql "$DATABASE_URL" -c \
  "SELECT topic FROM event_outbox WHERE topic='deal.partner_assigned' AND entity_id='${DEAL_ID}'
   ORDER BY id DESC LIMIT 1;"
```

Expected: one row with `topic='deal.partner_assigned'`.

## Step 7: List deals filtered by `partner_org_id`

The deal was created with org A and later changed to org B. The filter must return it by its current
attribution (org B).

```bash
curl -i -sS \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  "${API_BASE}/deals?partner_org_id=${PARTNER_ORG_B_ID}"
```

Expected: `200 OK`. The `data` array contains the deal with `id=${DEAL_ID}`.

Confirm the old org A filter no longer matches this deal:

```bash
curl -sS \
  -H "X-Workspace-Id: ${WS_ID}" \
  -H "X-User-ID: ${ADMIN_USER_ID}" \
  "${API_BASE}/deals?partner_org_id=${PARTNER_ORG_ID}" \
  | grep -c "${DEAL_ID}" || true
```

Expected: `0` — the deal moved to org B and must not appear in an org A filter.
