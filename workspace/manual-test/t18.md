## UAT 3 — Contacts list: server-side strength sort

**Browser path**:

1. On the Contacts list (`/people`), click the "Strength" control in the toolbar.

Expected:
- The browser's network tab shows a request to `/people?sort=-strength`
  (or `sort=strength` depending on toggle state — the first click sorts
  strongest-first per the `-strength` convention).
- The row order in the rendered table changes: Alice (the only contact with a
  non-null strength) appears first; no-signal contacts sort last.
- Clicking "Strength" again reverses the order (weakest-first); a third click
  returns to the default order. The sort is driven by the server response, not a
  client-side re-sort — confirm by reading `data[]` order in the raw network
  response and comparing it to the rendered row order.

**API equivalent**:

```bash
curl -s "$API_BASE/people?sort=-strength" \
  -H "X-Workspace-Id: $WS_ID" \
  -H "X-User-ID: $ADMIN_ID" | jq '[.data[] | {full_name, score: .strength.score}]'
```

Expected: 200. Alice appears first with a non-null `score`; Bob and Carol appear after
with `score: null`.

```bash
curl -s "$API_BASE/people?sort=unknown_field" \
  -H "X-Workspace-Id: $WS_ID" \
  -H "X-User-ID: $ADMIN_ID" | jq '{status: .status, code: .code}'
```

Expected: 422 (Unprocessable Entity) with a `code` field indicating the sort value
is not allowed — not a 500, not a silent fallback to the default sort.
