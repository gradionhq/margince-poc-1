# GH-207 — Manual/Live UAT: FE issues in Epic 1

Boot: `make infra-up && make migrate-up && make seed-reset && make run`, then in a second
terminal `make fe-dev`. Log in as `admin@example.com` / `changeme` (or any seeded role).

1. **[live]** Open **People**. Expected: every avatar in the contacts list renders as a
   square/circle (equal width and height matching its size), never a stretched pill.
2. **[live]** Focus the **Search contacts...** box on the People or Companies list. Expected:
   the leading search icon sits visually centered inside the input's left padding, never outside
   the box or vertically off-center.
3. **[live]** From a company (or deal) detail page, click **New deal**. Expected: the modal
   renders centered in the viewport, never pinned to the top-left corner, regardless of where
   the trigger button sits on the page.
4. **[live]** Trigger any success/error/warning/info toast (e.g. an action on the pipeline board
   or company detail page that calls `pushToast`). Expected: the toast renders as a small,
   bottom-right corner notification (≈400px wide, 20×20px icon), never an oversized element
   overlapping page content.
5. **[auto]** `cd frontend && npx tsc --noEmit && npx vitest run --project unit && npx vitest run
   --project storybook`. Expected: all green — no regression to `Avatar`, `TextInput`, `Modal`,
   `Toast`, `UserAvatar`, `SearchField`, `ToastContainer` tests/stories.
6. **[auto]** `make check-fe-static`. Expected: exits 0 (biome + tsc + design-system/font/icon
   purity).
7. **[auto]** `make fe-build`. Expected: exits 0 — the `@source` fix (or its fallback) survives a
   production build, not just Vite's dev-server HMR path.
8. **[auto]** `grep -nF '@source "../../node_modules/@shared/ui/src"' frontend/src/app/index.css`.
   Expected: exactly one match, confirming the directive still names the vendored `@shared/ui`
   package's source path (not just that some `@source` line exists).
