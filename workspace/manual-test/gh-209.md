# GH-209 — Manual/Live UAT: tenant isolation (RLS) + `record_grant` sharing

Run this only after the stack is up with `infra-up -> migrate-up -> seed-reset -> run`. The commands
below assume:

- the API is listening on `http://localhost:8080`
- the database is reachable through `DATABASE_URL`
- `psql`, `curl`, and Python 3 are on `PATH` (Python 3 mints `X-Approval-Token` values in Step 5, the
  same convention `workspace/manual-test/t12.md` uses)
- you can act as a tenant by sending `X-Workspace-ID` + `X-User-ID` (the dev-header path
  `cmd/api/routes.go`'s `workspaceWrap` reads when no session cookie/passport bearer token is
  present), and as an agent by sending `Authorization: Bearer <passport token>`

Use one shell session so the variables exported below stay available for later steps. Every step is
independent of whether Tasks 1-8 landed as separate commits — run this at the FINAL GATE once the
whole branch is merged into one working tree.

## Prerequisites: `APPROVAL_TOKEN_SIGNING_SECRET`

Step 5 mints an `X-Approval-Token`. The signing secret must be set **both** in the shell running
`make run` (so the server can verify it) and in this UAT shell (so the token-minting script derives
the same per-workspace key). Add it to `.env.local` before starting the server:

```bash
# in .env.local (or export before make run)
APPROVAL_TOKEN_SIGNING_SECRET=dev-gh209-uat-secret
```

Then in this UAT shell:

```bash
export APPROVAL_TOKEN_SIGNING_SECRET=dev-gh209-uat-secret
```

```bash

curl -sS -o /tmp/gh209_share_noauth.json -w 'HTTP_STATUS=%{http_code}\n' -X POST \
  -H "Content-Type: application/json" -H "Authorization: Bearer ${AGENT_TOKEN}" \
  -d "{\"record_type\":\"deal\",\"record_id\":\"${DEAL_A_ID}\",\"subject_type\":\"user\",\"subject_id\":\"${USER_Z_ID}\",\"access\":\"read\"}" \
  "${API_BASE}/record-grants"

```

Expected: `HTTP_STATUS=403`, body `{"status":403,"code":"approval_required"}`.

```bash
# Mint an X-Approval-Token bound to this exact share_record operation.
export APPR_TOKEN=$(python3 - "${DEAL_A_ID}" "${USER_Z_ID}" "read" "gh209-jti-${DEAL_A_ID}" <<'PYEOF'
import hmac as _hmac, hashlib, base64, json, os, sys
from datetime import datetime, timezone, timedelta

def b64u(b):
    return base64.urlsafe_b64encode(b).rstrip(b'=').decode()

root = os.environ['APPROVAL_TOKEN_SIGNING_SECRET']
ws_id = os.environ['WS_A']
record_id, subject_id, access, jti = sys.argv[1], sys.argv[2], sys.argv[3], sys.argv[4]

# Per-workspace key: HMAC-SHA256(root, workspace_id) — matches crmapprovals.signingKey.
key = _hmac.new(root.encode(), ws_id.encode(), hashlib.sha256).digest()

# diff_hash must match crmapprovals.HashDiff's exact field set for createRecordGrant:
# record_type, record_id, subject_type, subject_id, access (encoding/json sorts keys).
diff = {'record_type': 'deal', 'record_id': record_id, 'subject_type': 'user',
        'subject_id': subject_id, 'access': access}
diff_hash = b64u(hashlib.sha256(
    json.dumps(diff, sort_keys=True, separators=(',', ':')).encode()
).digest())

exp = (datetime.now(timezone.utc) + timedelta(minutes=5)).strftime('%Y-%m-%dT%H:%M:%SZ')
claims = {'jti': jti, 'approval_id': 'uat-' + jti, 'workspace_id': ws_id,
          'tool': 'share_record', 'diff_hash': diff_hash, 'exp': exp, 'single_use': True}

hdr = b64u(json.dumps({'alg': 'HS256', 'typ': 'JWT'}, separators=(',', ':')).encode())
pay = b64u(json.dumps(claims, separators=(',', ':')).encode())
sig = _hmac.new(key, (hdr + '.' + pay).encode(), hashlib.sha256).digest()
print(hdr + '.' + pay + '.' + b64u(sig))
PYEOF
)
echo "APPR_TOKEN=${APPR_TOKEN}"

# Retry with the valid token.
curl -sS -o /tmp/gh209_share_authed.json -w 'HTTP_STATUS=%{http_code}\n' -X POST \
  -H "Content-Type: application/json" -H "Authorization: Bearer ${AGENT_TOKEN}" \
  -H "X-Approval-Token: ${APPR_TOKEN}" \
  -d "{\"record_type\":\"deal\",\"record_id\":\"${DEAL_A_ID}\",\"subject_type\":\"user\",\"subject_id\":\"${USER_Z_ID}\",\"access\":\"read\"}" \
  "${API_BASE}/record-grants"
cat /tmp/gh209_share_authed.json
```

Expected: `HTTP_STATUS=201`, body has `"record_type":"deal"`, `"subject_id":"${USER_Z_ID}"`,
`"access":"read"`, `"granted_by":"${ADMIN_A_ID}"` (the passport's granting human — `Decider`'s
`GrantedBy` per `crmctx.Principal.UserID`).
