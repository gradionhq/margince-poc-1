# T08 Manual UAT — relationship + stakeholder edges

Prereqs: `make infra-up && make migrate-up && make seed-reset && make run`. All requests carry
`X-Workspace-ID: <seeded workspace id>`, `X-User-ID: <seeded user id — e.g. the seeded admin,
00000000-0000-0000-0010-000000000001>`, and `Content-Type: application/json`. Substitute real
seeded `person_id`/`organization_id`/`deal_id` values from `make seed-reset`'s output.

## 1. Create an employment relationship → visible via listRelationships

    curl -s -X POST localhost:8080/relationships \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" \
      -d '{"kind":"employment","person_id":"'"$PERSON_A"'","organization_id":"'"$ORG_A"'","role":"vp_engineering","is_current_primary":true,"source":"ui","captured_by":"human:test"}'

Expected: `201 Created`, `Location: /relationships/<id>`, body has `id`, `kind":"employment"`,
`person_id`/`organization_id` matching the request, `version: 1`.

    curl -s "localhost:8080/relationships?organization_id=$ORG_A&kind=employment" -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001"

Expected: `200 OK`, `data` array contains the row just created.

## 2. Second current-primary employment for the same person → confirms the chosen PO-AC-12 behavior

    curl -s -X POST localhost:8080/relationships \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" \
      -d '{"kind":"employment","person_id":"'"$PERSON_A"'","organization_id":"'"$ORG_B"'","is_current_primary":true,"source":"ui","captured_by":"human:test"}'

Expected: `409 Conflict` (`code: "conflict"`) — the chosen behavior is reject-not-auto-demote (see
`RelationshipStore.Create`'s doc comment). The first employment row from Step 1 remains fetchable
and unchanged via `GET /relationships?person_id=$PERSON_A&kind=employment`.

## 3. Two deal_stakeholder relationships, different roles → both visible; duplicate role → 409

    curl -s -X POST localhost:8080/relationships \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" \
      -d '{"kind":"deal_stakeholder","deal_id":"'"$DEAL_A"'","person_id":"'"$PERSON_A"'","role":"influencer","source":"ui","captured_by":"human:test"}'
    curl -s -X POST localhost:8080/relationships \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" \
      -d '{"kind":"deal_stakeholder","deal_id":"'"$DEAL_A"'","person_id":"'"$PERSON_B"'","role":"blocker","source":"ui","captured_by":"human:test"}'

Expected: both `201 Created`.

    curl -s "localhost:8080/deals/$DEAL_A/stakeholders" -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001"

Expected: `200 OK`, `data` has 3 rows: the 2 just created (influencer + blocker) plus one
pre-existing B-E09.9 seed fixture (role `champion`) — this guide's step is proven by the presence
of the influencer+blocker rows, not an exact count of 2.

    curl -s -X POST localhost:8080/relationships \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" \
      -d '{"kind":"deal_stakeholder","deal_id":"'"$DEAL_A"'","person_id":"'"$PERSON_A"'","role":"influencer","source":"ui","captured_by":"human:test"}'

Expected: `409 Conflict` (duplicate `deal_id`+`person_id`+`role`).

## 4. Stale If-Match on update → 409 version_skew

    RID=<id of the influencer stakeholder row from Step 3>
    curl -s -X PATCH "localhost:8080/relationships/$RID" \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" -H "If-Match: 999" \
      -d '{"role":"economic_buyer"}'

Expected: `409 Conflict`, `code: "version_skew"`.

    curl -s -X PATCH "localhost:8080/relationships/$RID" \
      -H "X-Workspace-ID: $WS" -H "X-User-ID: 00000000-0000-0000-0010-000000000001" -H "Content-Type: application/json" -H "If-Match: 1" \
      -d '{"role":"economic_buyer"}'

Expected: `200 OK`, `role` updated, `version: 2`.
