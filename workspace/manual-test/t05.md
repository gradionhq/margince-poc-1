# T05 Manual Test Guide — Organization CRUD

Prereqs: real stack up (`make infra-up && make migrate-up && make seed-reset && make run`), a
workspace + `X-Workspace-ID` header (or seeded session) for every request below.

## 1. Create with domain normalization + audit/event

    curl -s -X POST localhost:8080/organizations \
      -H "X-Workspace-ID: $WS_ID" -H "Content-Type: application/json" \
      -d '{"display_name":"Acme Inc","domains":[{"domain":"Acme.COM","is_primary":true}],"source":"ui","captured_by":"human:'"$USER_ID"'"}'

Expected: `201`, response body's `domains[0].domain == "acme.com"`. Then:

    psql "$DATABASE_URL" -c "select count(*) from audit_log where entity_type='organization' and action='create' and entity_id='<id from response>';"
    psql "$DATABASE_URL" -c "select count(*) from event_outbox where topic='organization.created' and entity_id='<id from response>';"

Expected: both counts are `1`.

## 2. Duplicate domain 409

    curl -s -X POST localhost:8080/organizations \
      -H "X-Workspace-ID: $WS_ID" -H "Content-Type: application/json" \
      -d '{"display_name":"Second Co","domains":[{"domain":"acme.com"}],"source":"ui","captured_by":"human:'"$USER_ID"'"}'

Expected: `409`, `code: duplicate_domain`, `details.existing_id` equals the first org's id, no
`Second Co` row is left in `organization`.

## 3. Archived org stays fetchable

    curl -s -X DELETE localhost:8080/organizations/<id> -H "X-Workspace-ID: $WS_ID"
    curl -s localhost:8080/organizations/<id> -H "X-Workspace-ID: $WS_ID"

Expected: the `DELETE` returns `200` with `archived_at` set; the follow-up `GET` still returns `200`
(PO-AC-6), same `id`, same non-null `archived_at`.

## 4. If-Match concurrency

    curl -s -X PATCH localhost:8080/organizations/<id> -H "X-Workspace-ID: $WS_ID" \
      -H "If-Match: 999" -H "Content-Type: application/json" -d '{"display_name":"Renamed"}'

Expected: `409`, `code: version_skew`.

    curl -s -X PATCH localhost:8080/organizations/<id> -H "X-Workspace-ID: $WS_ID" \
      -H "If-Match: not-a-number" -H "Content-Type: application/json" -d '{"display_name":"Renamed"}'

Expected: `400`.

## 5. List filters

    curl -s "localhost:8080/organizations?classification=partner&relevance_gte=50" -H "X-Workspace-ID: $WS_ID"

Expected: `200`, `data` contains only live orgs with `classification == "partner"` and
`relevance >= 50`.
