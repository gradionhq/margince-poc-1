# T09 — Relationship strength (PO-F-3) — Manual/Live UAT Guide

Prereqs: `make infra-up && make migrate-up && make seed-reset && make run` (backend on its
configured port; use `$BASE_URL` and a valid `Authorization`/workspace header per this repo's
seeded auth fixtures — see `workspace/manual-test/README.md` or an existing guide for the exact
curl header shape this stack expects).

## 1. List sorted by strength (ascending + descending)

```
curl -s "$BASE_URL/people?sort=strength" -H "$AUTH_HEADERS" | jq '.data[] | {id, full_name, strength}'
```
Expected: 200. Every person row carries a `strength` key (object or `null`). Scores across
`data` are non-increasing — `sort=strength` (unprefixed) means strongest-first (descending) by
this ticket's deliberate convention (see `listByStrength`'s doc comment, Task 2 Step 5):
relationship strength reads naturally strongest-first, unlike an alphabetical/numeric key where
`-`=descending is the norm.

```
curl -s "$BASE_URL/people?sort=-strength" -H "$AUTH_HEADERS" | jq '[.data[].strength.score]'
```
Expected: 200, the reverse ordering of the previous call (for people with a non-null strength;
no-signal-yet people sort last regardless of direction).

## 2. Golden-example person detail

Seed (via `POST /activities` + linking, or the seed fixtures if `make seed-reset` already
includes a PO-F-3 worked-example person) a person with: last interaction 5 days ago, 12
interactions in the last 90 days, 7 inbound / 5 outbound.

```
curl -s "$BASE_URL/people/<id>" -H "$AUTH_HEADERS" | jq '.strength'
```
Expected: `{"score": 47, "bucket": "moderate", ...}`.

## 3. Strength breakdown evidence

```
curl -s "$BASE_URL/people/<id>/strength-breakdown" -H "$AUTH_HEADERS" | jq .
```
Expected: 200. `contributing_activities` lists exactly the 12 seeded interaction ids.
`recency`/`frequency`/`reciprocity` match the `.strength` block from step 2 exactly (same
person, same read-time computation).

## 4. Zero-interaction person

```
curl -s "$BASE_URL/people/<new-person-with-no-activity-id>" -H "$AUTH_HEADERS" | jq '.strength'
```
Expected: `null` — never a literal `0`.

## 5. Stale-only person (interactions exist, none within 90d)

Seed a person with one activity 120+ days old, nothing more recent.

```
curl -s "$BASE_URL/people/<id>" -H "$AUTH_HEADERS" | jq '.strength'
```
Expected: `{"score": 0, "bucket": "weak", "no_recent_activity": true, ...}`.
