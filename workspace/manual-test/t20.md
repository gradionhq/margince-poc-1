# T20 live-UAT — Company 360

Prereqs: `make infra-up && make migrate-up && make seed-reset && make run` (backend); `pnpm --dir
frontend run dev` (frontend). Log in as a seeded rep. Seed must include: (a) at least one org with
a computed `org_strength` (multiple contacts, one clearly strongest), (b) one org with zero
computed contact strength (`org_strength: null`), (c) one org that IS a partner (has a `partner`
row) and one that is NOT, (d) an org with an open deal whose stakeholder has `role: champion`, (e)
an org with a stalled deal and an org with none, (f) an org with domains/address/size_band set and
one with all of those null.

1. **Row click-through.** From `/companies`, click any company row.
   Expected: navigates to `/companies/{id}`; the row's `onClick` (previously unwired) now fires.
2. **Header card, full fields.** Open the org with domains/address/size_band set.
   Expected: `OrgLogo` (monogram, `size="lg"`), `display_name`, industry, a website link
   (`https://{primary domain}` from `organization.domains`, opens new tab), banded staff
   (`size_band`), location (`"{city}, {country}"`).
3. **Header card, honest omission.** Open the org with all those fields null.
   Expected: each null meta-row field is simply absent — no "N/A", no placeholder text.
4. **Org strength card, signal present.** Open the org with a computed `org_strength`.
   Expected: "Account relationship strength" card shows `{score}/100`, a "computed · MAX over
   contacts" chip, a fill bar, a source line naming the strongest contact
   (`org_strength.top_person_name`) linking to `/people/{top_person_id}`, and copy using "not an
   average, not a black box" language (not "capped/normalized/recency decay" — that copy is
   corpus-flagged drift, must not appear).
5. **Org strength expansion.** Click "Show the per-contact scores behind this".
   Expected: expands a per-contact list, one row per fetched contact, each showing that contact's
   own score; the org's top-line score equals the max of the expanded list (not an average); the
   trigger label flips to a collapse state; clicking again collapses it.
6. **Org strength, no-signal.** Open the zero-signal org.
   Expected: an honest "no signal yet" card (STATE-1), not a zeroed/blank card.
7. **People rail.** On the org with a champion-flagged contact.
   Expected: each contact row shows avatar, name, title, a role flag (Champion vs
   Stakeholder/none), and `{score}/100` (or "no signal yet"); the top-scoring contact is visually
   highlighted; each row links to `/people/{person_id}`.
8. **Deals rail — org's own deals.** Same org, if it has deals.
   Expected: open+won deals listed (no lost deals), each with name, amount (`Intl.NumberFormat`,
   the deal's own currency), and a stage/stalled/single-threaded pill; each row links to
   `/deals/{deal_id}` (falls through to the app's catch-all until T21 merges — expected, not a
   bug).
9. **Activity card provenance.** Any org with activity.
   Expected: each row carries a `SourceChip`; footer reads "You logged none of this — capture
   linked every item".
10. **Account-signal card, stalled cited.** Open the org with a stalled deal.
    Expected: "single-threaded"/stalled sub-line cites that specific deal with an "open the deal"
    link.
11. **Account-signal card, honest omission.** Open the org with no stalled deal.
    Expected: that sub-line is entirely absent — never a fabricated "no signal" placeholder line
    (STATE-5).
12. **Quick facts rail.** Any org.
    Expected: Owner, Open deals (`open_deal_count`), Won lifetime (per-currency, never a mixed
    total), People known (`contact_count`), First seen (`created_at`), Last touch (latest
    activity, else `updated_at`).
13. **Partner panel present.** Open the org that IS a partner.
    Expected: `cert_status`/`partner_role`/`margin_tier` render, plus a "deals it sourced" slice
    distinct from the deals rail (partner_org_id-attributed, not this org's own deals).
14. **Partner panel absent (STATE-1, not STATE-3).** Open the org that is NOT a partner.
    Expected: no partner panel at all — no error card, no skeleton stuck loading, nothing.
15. **Edit marks typed-by-you.** Click Edit, change a field, save.
    Expected: the changed field(s) show a 🟢 typed-by-you badge.
16. **New deal modal.** Click "New deal".
    Expected: modal opens staging a deal with this org and its known contacts pre-linked; no
    network call fires until Confirm.
17. **Summarize this account, disabled-honest.** Locate the "Summarize this account" button.
    Expected: visible, disabled, with a tooltip/caption explaining why (not a fake summary, not a
    live call).
18. **Per-card failure isolation (STATE-3).** Simulate the people-rail's N+1 partially failing
    (e.g. throttle/kill one `getPerson` call in dev tools).
    Expected: that one contact row shows a cause+retry, the rest of the screen stays intact — no
    full-screen blank.
