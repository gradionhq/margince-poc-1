# T10 Live UAT Guide

Run this only after the stack is up with `infra-up -> migrate-up -> seed-reset -> run`.
The commands below assume:

- the API is listening on `http://localhost:8080`
- the database is reachable through `DATABASE_URL`
- you can act as a tenant by sending `X-Workspace-Id` + `X-User-ID`

Use one shell session so the variables set in the bootstrap block stay available for the later steps.

## Bootstrap

```bash
export API_BASE='http://localhost:8080'
export WS_ID='00000000-0000-0000-0000-000000000030'
export USER_ID='55555555-5555-5555-5555-555555555561'
export ROLE_ID='66666666-6666-6666-6666-666666666671'
export PIPELINE_A='11111111-2222-3333-4444-555555555561'
export PIPELINE_B='11111111-2222-3333-4444-555555555562'

psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO workspace (id, name, slug, base_currency) VALUES ('${WS_ID}', 'T10 live UAT workspace', 't10-live-uat', 'USD') ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO pipeline (id, workspace_id, name, is_default, position) VALUES ('${PIPELINE_A}', '${WS_ID}', 'T10 Pipeline A', true, 1) ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO pipeline (id, workspace_id, name, is_default, position) VALUES ('${PIPELINE_B}', '${WS_ID}', 'T10 Pipeline B', false, 2) ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO stage (id, workspace_id, pipeline_id, name, position, semantic, win_probability) VALUES ('11111111-2222-3333-4444-555555555571', '${WS_ID}', '${PIPELINE_A}', 'New', 1, 'open', 10) ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO stage (id, workspace_id, pipeline_id, name, position, semantic, win_probability) VALUES ('11111111-2222-3333-4444-555555555572', '${WS_ID}', '${PIPELINE_A}', 'Qualified', 2, 'open', 25) ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO stage (id, workspace_id, pipeline_id, name, position, semantic, win_probability) VALUES ('11111111-2222-3333-4444-555555555573', '${WS_ID}', '${PIPELINE_A}', 'Closed Won', 3, 'won', 100) ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO stage (id, workspace_id, pipeline_id, name, position, semantic, win_probability) VALUES ('11111111-2222-3333-4444-555555555574', '${WS_ID}', '${PIPELINE_A}', 'Closed Lost', 4, 'lost', 0) ON CONFLICT (id) DO NOTHING;"
export STAGE_QUALIFIED='11111111-2222-3333-4444-555555555572'

psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO app_user (id, workspace_id, email, display_name) VALUES ('${USER_ID}', '${WS_ID}', 't10-uat@example.com', 'T10 UAT User') ON CONFLICT (id) DO NOTHING;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO role (id, workspace_id, key, is_system, permissions) VALUES ('${ROLE_ID}', '${WS_ID}', 't10_admin', true, '{\"pipeline\":{\"read\":{\"row_scope\":\"all\"},\"update\":{\"row_scope\":\"all\"}},\"stage\":{\"read\":{\"row_scope\":\"all\"},\"update\":{\"row_scope\":\"all\"}}}') ON CONFLICT (workspace_id, key) DO UPDATE SET permissions = EXCLUDED.permissions;"
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "INSERT INTO role_assignment (workspace_id, role_id, user_id) VALUES ('${WS_ID}', '${ROLE_ID}', '${USER_ID}') ON CONFLICT (role_id, user_id, COALESCE(team_id, '00000000-0000-0000-0000-000000000000'::uuid)) DO NOTHING;"
```

## Step 1: List pipelines and get the pipeline with its ordered stages

```bash
curl -sS -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" "${API_BASE}/pipelines" | sed -n '1,200p'
curl -i -sS -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" "${API_BASE}/pipelines/${PIPELINE_A}"
```

Expected: `GET /pipelines` returns `200` with both pipelines; `GET /pipelines/{id}` returns `200` with the seeded pipeline and its four stages ordered New(1)/Qualified(2)/Closed Won(3)/Closed Lost(4).

## Step 2: Attempt to make Pipeline B a second default

```bash
curl -i -sS -X PATCH \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" \
  -d '{"is_default":true}' \
  "${API_BASE}/pipelines/${PIPELINE_B}"
```

Expected: `409 Conflict`.

## Step 3: Move "Qualified" to the position "Closed Won" already holds

```bash
curl -i -sS -X PATCH \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" \
  -d '{"position":3}' \
  "${API_BASE}/stages/${STAGE_QUALIFIED}"
```

Expected: `409 Conflict`.

## Step 4: Attempt to retune the "Closed Won" terminal stage's probability

```bash
curl -i -sS -X PATCH \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" \
  -d '{"win_probability":90}' \
  "${API_BASE}/stages/${STAGE_WON}"
```

Expected: `422 Unprocessable Entity`.

## Step 5: Retune "Qualified" from 25 to 30 and re-read it live

```bash
curl -i -sS -X PATCH \
  -H "Content-Type: application/json" \
  -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" \
  -d '{"win_probability":30}' \
  "${API_BASE}/stages/${STAGE_QUALIFIED}"

curl -sS -H "X-Workspace-Id: ${WS_ID}" -H "X-User-ID: ${USER_ID}" "${API_BASE}/stages/${STAGE_QUALIFIED}"
```

Expected: both responses show `win_probability: 30` — the second (plain re-GET) proves the value is read live, not cached.

## Step 6: Concurrent-reorder — automated, not manual [auto]

Covered by `TestStageStore_Update_ConcurrentSwap_NoUniquenessViolationWindow` in `backend/internal/modules/deals/concurrent_reorder_test.go` (Task 3) — this is a race-condition property best proven by two real concurrent goroutines against the test DB, not by two manually-timed `curl` calls in a guide. Run: `cd backend && TEST_DATABASE_URL="$DATABASE_URL" go test -tags=integration ./internal/modules/deals/... -run TestStageStore_Update_ConcurrentSwap_NoUniquenessViolationWindow -v -count=5`.

Expected: `PASS` on all 5 runs.
