# T21 live-UAT — pipeline board + table

Prereqs: `make infra-up && make migrate-up && make seed-reset && make run` (backend); `pnpm --dir
frontend run dev` (frontend). Log in as a seeded rep with an `owner`/`member` role that has at
least one open deal per stage plus one stalled and one single-stakeholder deal (seed fixture must
provide this — if the default seed lacks stalled/single-threaded deals, add them to
`scripts/seed`/dev-seed fixtures as part of this task).

1. **Board loads grouped by stage.** Navigate to `/deals`.
   Expected: five columns — New (10%), Qualified (25%), Discovery (40%), Proposal (60%),
   Negotiation (80%) — each with a card count and a raw + weighted sub-line; no Won/Lost columns
   visible.
2. **Totals strip matches hand arithmetic.** Read the "Weighted", "Raw pipeline", "Open deals"
   values in the toolbar.
   Expected: Raw pipeline = sum of all visible card amounts (own currency converted to workspace
   base per DEAL-FORM-2); Weighted = sum of `amount × stage probability` per card, rounded half
   away from zero; the two numbers reconcile to the per-column sub-lines.
3. **Drag a card between two open stages.** Drag a card from "Qualified" to "Discovery".
   Expected: drop target column highlights while hovering; on drop the card appears immediately in
   Discovery (optimistic) with its age reset to "0d"; a toast reads "<Company> moved Qualified →
   Discovery · stage history recorded"; the totals strip re-fetches and updates.
4. **Click without drag navigates; a drag tail does not.** Click a card without moving the mouse.
   Expected: navigates to `/deals/:id`. Go back, then drag a card a few pixels and release without
   crossing a column boundary — expected: it does NOT navigate.
5. **A drop that the server refuses snaps back.** (Requires a way to force a refusal — e.g. drag a
   deal you don't own into a stage requiring an owner change your role forbids, or stop the backend
   mid-drag to force a network error.)
   Expected: the card returns to its original column; a toast names the specific cause (not a
   generic "failed"); the column re-fetches.
6. **Stalled + single-threaded flags render.** Locate the seeded stalled deal and the seeded
   single-stakeholder deal.
   Expected: stalled card shows a left orange border + amber "Stalled Nd" flag with a hover title;
   single-stakeholder card shows a red "Single-threaded" flag with a hover title.
7. **Advance button, open→open.** Open a card's Advance action while it's in an open stage with
   another open stage after it.
   Expected: advances directly to the next stage with no confirm dialog.
8. **Advance/drag into Won.** Advance (or drag) a card into the transient Won drop zone (visible
   only during an active drag) / click Advance when the next stage is terminal.
   Expected: outcome confirm dialog opens; click OK.
   Expected: deal shows Closed Won, weighted value now reflects 100%, card leaves the open board.
9. **Advance into Lost, blank reason.** Repeat for a different deal targeting Lost; in the dialog
   click Cancel/the lost-reason path but submit a blank reason.
   Expected: the deal stays open, nothing changes (no stage change, no toast, no server call
   completed).
10. **Advance into Lost, with a reason.** Repeat, this time typing a non-blank lost reason and
    submitting.
    Expected: deal shows Closed Lost with that reason; weighted value now 0%.
11. **Backward open→open shows no confirm.** Drag a card from a later open stage back to an
    earlier open stage (e.g. Negotiation → Discovery).
    Expected: no confirm dialog — the move applies directly (DEAL-AC-N-1).
12. **Board/Table toggle preserves state.** Apply any active filter (if present), switch to Table,
    confirm the same deals/filter persist as sortable rows (Company, Deal, Stage, Prob., Amount,
    Weighted, Age) sorted by amount desc; switch back to Board.
    Expected: no reload, no filter loss, both views agree on the same deal set.
13. **Table stalled cell.** In Table view, find the stalled deal's row.
    Expected: its Age cell is amber with a warning icon.
14. **New deal from context.** From a company or contact's page (or the board's "New deal" button
    with a company selected in context), click New deal.
    Expected: modal opens with the company pre-linked, a suggested deal name, stakeholders
    pre-attached, default pipeline/first stage selected, and a recent-activity count. If that org
    already has an open deal, a duplicate warning is shown.
15. **Confirm & create.** Click "Confirm & create".
    Expected: success toast, modal closes, new card appears on the board in the first open stage.
16. **Cancel makes no change.** Reopen New deal, fill fields, click Cancel.
    Expected: modal closes; no new card appears; no network POST to `createDeal` fired (check dev
    tools network tab: zero requests to `/deals` POST during this step).
17. **Empty board is honest.** (Requires a workspace/dataset with zero deals — a fresh seed or a
    filter that matches nothing.)
    Expected: board renders an explicit "No deals yet" / "No deals match" empty state — never a
    blank screen or a spinner that never resolves; empty stage columns still render as valid drop
    targets (not collapsed).
18. **Loading and error states.** Throttle network to observe the loading state; then stop the
    backend and reload `/deals`.
    Expected: skeleton/loading chrome renders immediately, content streams in; on a backend error,
    an honest error card with retry appears (not a blank page).
