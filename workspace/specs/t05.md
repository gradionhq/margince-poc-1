# T05 — Backend — organization module (list/get/create/update/archive) with domain-collision 409 and real classification/relevance columns

## Goal

Complete the `/organizations` CRUD surface. `listOrganizations` already exists
(`backend/internal/modules/directory/transport/handler_org.go`, shipped by T18); this ticket
adds `createOrganization`, `getOrganization`, `updateOrganization`, `archiveOrganization`,
wires the store's already-present `OrgStore.Create/Get/Update/Archive` methods to HTTP, adds
the domain-collision 409, the `classification`/`relevance` DB-level correctness fix, the
missing audit/event floor, and the `classification`/`relevance` list filters.

## Discovered current state (read before implementing — corrects the ticket body's stated layout)

- **Handler package is `backend/internal/modules/directory/transport`, not
  `modules/people/transport`.** `handler_org.go` (list-only, 49 lines) and `handler_deal.go`
  already live there (T11/T18); routes.go already wires
  `dealtransport.NewOrganizationHandler(...)` to `/organizations` via the shared `crud()`
  helper (`backend/cmd/api/routes.go` ~L118). Add the new methods to this package — do not
  create a new one, do not touch `modules/people/transport`.
- `backend/internal/modules/directory/store_org.go` (`OrgStore`) already implements
  `Create`, `Get`, `List`, `Update`, `Archive`, `getAny` — but:
  - `Create` does not touch `organization_domain` at all (no domain read from the request,
    no collision check, no insert) and writes **no `audit_log` row and no outbox event**.
  - `Update` and `Archive` also write **no `audit_log` row and no outbox event** (contrast
    `PersonStore` in `store.go` ~L178/339/362, which does this correctly per entity — mirror
    that, not `OrgStore`'s current state).
  - `Update` only touches `name`/`website`/`owner_id` — no domain patch path is required by
    the acceptance criteria below (domains are their own contract shape); leave as-is.
  - `List` filters only by `sort`; no `classification`/`relevance_gte`/`domain`/`owner_id`/`q`
    despite `crm.yaml` already declaring all of them on `listOrganizations`.
- Schema: `organization.classification` is `text NULL` (migration `000003_core_objects`),
  not `NOT NULL DEFAULT 'prospect'` per PO-DDL-4. `organization_domain` (workspace-scoped,
  `uq_org_domain UNIQUE(workspace_id, domain)`, `domain = lower(domain)` CHECK) and
  `idx_org_class`/`relevance` already exist (migration `000006_org_gaps_person_phone`) — no
  new table needed, only the classification column tightening.
- `backend/api/crm.yaml` already fully declares `listOrganizations`, `createOrganization`
  (`CreateOrganizationRequest` incl. `domains[]`, 409 `duplicate_domain` with
  `details.existing_id`), `getOrganization`, `updateOrganization`
  (`UpdateOrganizationRequest`), `archiveOrganization` — this ticket is transport + store +
  one migration, **no contract change**.

## Acceptance criteria (build-verifiable)

1. Migration: `organization.classification` becomes `NOT NULL DEFAULT 'prospect'`
   (backfill existing NULL rows to `'prospect'` before adding the constraint) — PO-DDL-4.
   `relevance smallint 0-100 nullable` already correct; do not touch. (verify: integration)
2. `createOrganization`: inserts the org, normalizes (`strings.ToLower`, trim) each
   `domains[].domain`, and — inside the same workspace-scoped tx as the org insert — checks
   each normalized domain against live (`archived_at IS NULL`) `organization_domain` rows
   for the workspace. Any collision → whole request fails, `409 duplicate_domain`,
   `details.existing_id` = the colliding row's `organization_id`, `details.field =
   "domains[0].domain"` (index of the first colliding entry). No collision → each domain
   row is inserted into `organization_domain` (first entry `is_primary=true` unless the
   request already marks one). (verify: integration)
3. `getOrganization`, `updateOrganization` (PATCH-merge, `If-Match` version concurrency
   mirroring `handler_deal.go`'s `update()` idiom: malformed header → 400, stale version →
   `409 version_skew`), `archiveOrganization` (soft-delete; archived orgs remain
   fetchable by id per PO-AC-6 — `OrgStore.getAny` already supports this, wire
   `getOrganization` to use `Get` and confirm `archiveOrganization`'s response uses
   `getAny`). Each maps store errors (`ErrNotFound` → 404, `ErrVersionSkew` → 409) via the
   package's existing `jsonErr`/`jsonProblem` helpers — do not duplicate error-mapping
   logic already in `handler_deal.go`. (verify: integration)
4. `listOrganizations` gains `classification` (exact match) and `relevance_gte` (`relevance
   >= N`) filters per DM-VOCAB-2, plus `domain` (exact normalized-domain lookup via
   `organization_domain`) and `owner_id` — all four already declared as query params in
   `crm.yaml`. Sort vocabulary (`id`, `strength`, `-strength`) is unchanged. (verify:
   integration)
5. PO-AC-3 / GATE-CORE-3/5: every `OrgStore` mutation (`Create`, `Update`, `Archive`)
   writes exactly one `audit_log` row via `crmaudit.WriteTx` (mirror `PersonStore` in
   `store.go`) and enqueues exactly one `event_outbox` row (`organization.created`,
   `organization.updated`, `organization.archived` — mirror `store_deal.go`'s
   `deal.created`/outbox pattern) in the same tx as the mutation. `source`/`captured_by`
   stay non-null on every row (already enforced by `requireProvenance` in `Create`; do not
   relax it). (verify: integration)
6. `make check` is green; no regression in existing `directory`/`transport` package tests
   (`handler_org_test.go`'s existing list-only tests must keep passing). (verify: command)

1. `POST /organizations` with `domains: [{domain: "Acme.COM", is_primary: true}]`.
   Expected: 201, response's `domains[0].domain == "acme.com"`, one `audit_log` row
   (action `create`), one `organization.created` outbox row.
2. `POST /organizations` again with a domain already owned by a live org in the same
   workspace. Expected: `409`, problem `code: duplicate_domain`,
   `details.existing_id` = the first org's id; no partial org/domain row left behind.

   malformed header → `400`.
5. `GET /organizations?classification=partner&relevance_gte=50` → only matching live orgs,
   correctly filtered.
6. `make check` exits 0; `make test-integration` shows 0 skips on the `directory`/`transport`
   packages.
