**Goal:** Land a single reversible migration pair that creates the `custom_field` catalog table
(CUSTOM-FIELDS-SCHEMA-1) — the system-of-record for every runtime-added column. Base
columns + version + RLS per the universal conventions (DM-CONV-3/4/5-8), two unique indexes, no
options sidecar table (options live in the `options` jsonb column). This migration creates
**no** `cf_`-prefixed columns on any object table (that engine work is CF-T03), no Go
handlers/service/repo code, no contract changes (CF-T01, already merged, independent).

**Scope:** be-only. The diff touches only `backend/migrations/000071_custom_field_catalog.{up,down}.sql`
and `workspace/manual-test/cf-t02.md` — no frontend, no contract, no handler code. Per-task
type/build gate = `go build ./backend/...` (a migration file itself isn't Go, but the
`migrate` integration test package must still compile) — no `make fe-typecheck` needed. FINAL
GATE keeps `make check-q`; FINAL-UAT coverage keeps; live-stack UAT runs the full stack (schema
change, needs a real Postgres); craftsmanship review is the full agent (per the scope-aware
matrix, be-only still gets the full live-stack UAT and full craftsmanship agent — only the
fe-uat capture lane is skipped, and there's no FE surface to skip it from).

**Architecture:** One additive migration pair, no changes to any existing table. Mirrors
`backend/migrations/000069_record_grant.up.sql`/`.down.sql` in shape (base columns, version,
`ENABLE`/`FORCE ROW LEVEL SECURITY`, the `_tenant_isolation` policy, `GRANT` statement, clean
down) but is simpler — a single new tenant-scoped table with no widening of any other table's
RLS policy (`record_grant`'s widening logic is specific to that ticket, not reused here).
Uses `touch_versioned()` (defined in `000001_init.up.sql`, bumps both `updated_at` and
`version` in one `BEFORE UPDATE` trigger — the DM-CONV-4 pattern used by every mutable core
table, e.g. `trg_lead_touch` in `000003_core_objects.up.sql`), not the bare `set_updated_at()`
(which record_grant doesn't even use, since it has no `updated_at`/`archived_at` columns —
`record_grant` is a partial exemplar for RLS shape only, not the full base-column set).

**Tech Stack:** `golang-migrate` (`backend/migrations/`, flat-numbered `NNNNNN_slug.{up,down}.sql`),
applied via `make migrate-up`/`make migrate-down`, exercised in CI by
`backend/internal/shared/ports/migrate/migrate_integration_test.go` (loads the whole
`backend/migrations` dir through `RunWith`, so this migration is applied as part of every
`make test-integration` run).

- [ ] **Step 2: Write the up migration**

Create `backend/migrations/000071_custom_field_catalog.up.sql`:

```sql
-- 000071 — custom_field catalog (CUSTOM-FIELDS-SCHEMA-1): system-of-record for every
-- runtime-added column. Base columns (DM-CONV-3) + version (DM-CONV-4) + RLS (DM-CONV-5-8),
-- mirroring 000069_record_grant's tenant_isolation shape (that migration itself has no
-- updated_at/archived_at/version-trigger, so the trigger here instead mirrors
-- 000003_core_objects's trg_lead_touch). No cf_-prefixed columns land here — that engine
-- work is a later ticket (CF-T03). No options sidecar table — picklist values live in the
-- `options` jsonb column (custom-fields.md, "no separate options table in V1").
--
-- DEVIATION (documented): no CHECK ties `currency` to `type = 'currency'` — the spec asks
-- for that conditional-required validation to stay an application-layer (handler) concern
-- rather than a non-trivial cross-column DDL expression.

CREATE TABLE custom_field (
  id            uuid PRIMARY KEY DEFAULT uuidv7(),
  workspace_id  uuid NOT NULL REFERENCES workspace(id) ON DELETE RESTRICT,
  object        text NOT NULL CHECK (object IN ('person','organization','deal','lead','activity')),
  slug          text NOT NULL,
  label         text NOT NULL,
  type          text NOT NULL CHECK (type IN ('text','number','date','currency','picklist','boolean')),
  status        text NOT NULL DEFAULT 'active' CHECK (status IN ('active','retired')),
  column_name   text NOT NULL,
  currency      char(3) NULL CHECK (currency IS NULL OR currency ~ '^[A-Z]{3}$'),
  options       jsonb NULL,
  created_by    uuid NOT NULL REFERENCES app_user(id),
  created_at    timestamptz NOT NULL DEFAULT now(),
  updated_at    timestamptz NOT NULL DEFAULT now(),
  archived_at   timestamptz NULL,
  version       bigint NOT NULL DEFAULT 1
);

CREATE UNIQUE INDEX idx_custom_field_slug ON custom_field (workspace_id, object, slug);
CREATE UNIQUE INDEX idx_custom_field_col  ON custom_field (workspace_id, object, column_name);

CREATE TRIGGER trg_custom_field_touch BEFORE UPDATE ON custom_field
  FOR EACH ROW EXECUTE FUNCTION touch_versioned();

ALTER TABLE custom_field ENABLE ROW LEVEL SECURITY;
ALTER TABLE custom_field FORCE ROW LEVEL SECURITY;

CREATE POLICY custom_field_tenant_isolation ON custom_field
  USING      (workspace_id = nullif(current_setting('app.workspace_id', true), '')::uuid)
  WITH CHECK (workspace_id = nullif(current_setting('app.workspace_id', true), '')::uuid);

GRANT SELECT, INSERT, UPDATE, DELETE ON custom_field TO margince_app;
```
